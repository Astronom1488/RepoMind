# RepoMind — RAG assistant for developer tasks

## Goal

Build a portfolio-ready AI project for an AI-assisted developer internship.

The project should help a developer understand an unfamiliar codebase, find relevant files, explain logic, and suggest changes for bugfixes or small features.

## Core idea

RepoMind ingests a repository, indexes code and documentation, and answers questions in two modes:

1. **Ask mode** — general questions about the codebase.
2. **Fix mode** — issue-oriented analysis with suggested files, steps, risks, and tests.

## Why this project is strong

* Shows RAG fundamentals.
* Shows practical developer workflow use case.
* Demonstrates backend/API skills.
* Demonstrates product thinking, not just experimentation.
* Fits AI-assisted developer positioning better than a generic PDF chatbot.

## MVP features

### 1. Repository ingestion

* Load files from a local repository path.
* Support extensions:

  * `.py`
  * `.js`
  * `.ts`
  * `.md`
  * `.txt`
  * `.json`
* Ignore:

  * `.git`
  * `node_modules`
  * `venv`
  * `__pycache__`
  * build/dist folders

### 2. Chunking

* Markdown/text docs: chunk by paragraphs/sections.
* Code files: chunk by function/class when possible, otherwise sliding window.
* Store metadata:

  * file path
  * file type
  * language
  * chunk id

### 3. Embeddings and vector store

* Use `sentence-transformers/all-MiniLM-L6-v2` for embeddings.
* Use ChromaDB as vector store.

### 4. Retrieval

* Top-k semantic search.
* Return retrieved chunks with metadata.
* Basic deduplication by file path/chunk overlap.

### 5. LLM answer generation

Generate structured answers using retrieved context.

#### Ask mode output

* Short answer
* Relevant files
* Explanation
* Related code snippets

#### Fix mode output

* Problem summary
* Most relevant files
* Likely root cause
* Suggested change plan
* Risks
* Test checklist

### 6. API

FastAPI endpoints:

* `POST /index` — index a repository
* `POST /ask` — query Ask mode
* `POST /fix` — query Fix mode
* `GET /health`

### 7. UI

Streamlit interface with:

* input for repository path
* button to index repository
* question box
* mode switch: Ask / Fix
* result view
* retrieved files panel

## Example queries

### Ask mode

* Where is authentication handled?
* Which files are responsible for input validation?
* How does the API create a new user?

### Fix mode

* Endpoint `/login` returns 500 when email is missing. What files should I inspect?
* I need to add JWT expiration validation. Where should I start?
* A POST endpoint accepts invalid payloads. What should be changed?

## Interview value

This project lets you talk about:

* RAG pipeline design
* code/document chunking tradeoffs
* metadata-based retrieval
* developer-assistant UX
* practical AI workflow integration
* limitations of retrieval quality and hallucination control
